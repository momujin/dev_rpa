1. システム概要

本システムは、RPAツールの操作練習用として、ECサイトから出力された注文データをローカルの販売管理システムに転記するフローをシミュレートする環境です。

対応ECサイト

ECサイト モック画面 テーマカラー CSV形式
楽天RMS 店舗運営Navi風 #BF0000（赤） 約40カラム
Amazon セラーセントラル風 #FF9900（オレンジ） 約30カラム
Yahoo! ストアクリエイター風 #FF0033（赤） 注文+商品明細の2ファイル

プロジェクト構成

dev_rpa/
├── package.json                 # ルートパッケージ
├── mock-ec-site/                # モックECサイト
│   ├── index.html               # トップページ（サイト選択）
│   ├── css/                     # 各サイトのスタイル
│   ├── js/                      # データ生成・CSV出力
│   ├── rakuten/                 # 楽天RMS風画面
│   ├── amazon/                  # Amazonセラセン風画面
│   └── yahoo/                   # Yahoo!ストクリ風画面
├── sales-management/            # 販売管理システム（Electron）
│   ├── main.js                  # メインプロセス
│   ├── preload.js               # セキュアIPC
│   └── renderer/                # UI
└── sample-data/                 # サンプルCSVデータ

2. 環境構築・起動方法

前提条件

  • Node.js v18以上がインストールされていること
  • npmが使用可能であること

インストール

cd /Users/juumonjikouta/dev_web/dev_rpa
npm run install:all

起動コマンド

コマンド 説明 アクセス先
npm run dev 両方同時起動（推奨） Web + Electron
npm run start:web モックサイトのみ http://localhost:8080
npm run start:app 販売管理システムのみ Electronアプリ
ポイント: npm run dev を実行すると、モックECサイト（ブラウザ）と販売管理システム（Electron）が同時に起動します。

3. 演習1: CSVダウンロード → インポート

目的

基本的なWeb操作とデスクトップアプリ操作の習得

操作手順

Step 操作内容 対象要素（ID/属性）
1 http://localhost:8080 をブラウザで開く -
2 「楽天RMS」カードをクリック #link-rakuten
3 ナビゲーションで「データ分析」をクリック #nav-download
4 期間を「過去7日間」に設定 [data-rpa-field="date-range"]
5 出力件数を「20件」に設定 [data-rpa-field="order-count"]
6 「更新」ボタンでプレビュー生成 #btn-refresh-preview
7 「CSVダウンロード」ボタンをクリック #btn-download-csv
8 販売管理システムで「CSVインポート」を選択 #nav-import
9 インポート形式を「楽天RMS形式」に設定 [data-rpa-field="import-format"]
10 「参照...」でダウンロードしたCSVを選択 #btn-select-file
11 「インポート実行」をクリック #btn-import
12 インポート完了を確認 [data-status="success"]

4. 演習2: CSV読み取り → フォーム転記

目的

CSVファイルのデータを1件ずつ手動入力フォームに転記する操作の習得

シナリオ

ダウンロードしたCSVファイルをRPAツールで読み込み、販売管理システムの注文入力フォームに1行ずつデータを入力して保存します。

入力フォームの要素一覧

項目 data-rpa-field属性 入力タイプ
注文番号 order-id テキスト（必須）
注文日 order-date 日付（必須）
ECサイト ec-site セレクト（必須）
ステータス status セレクト
決済方法 payment-method セレクト
顧客名 customer-name テキスト（必須）
顧客名カナ customer-kana テキスト
郵便番号 postal-code テキスト
電話番号 phone テキスト
メールアドレス email テキスト
住所 address テキスト（必須）
商品コード item-code-0 テキスト
商品名 item-name-0 テキスト
数量 item-quantity-0 数値
単価 item-price-0 数値
送料 shipping-fee 数値
消費税 tax 数値
合計金額 total 数値

操作フロー

  1. RPAツールでCSVファイルを読み込む
  2. 販売管理システムで「注文入力」画面を開く（#nav-input）
  3. CSVの各行をループ処理
    • 各フィールドにデータを入力
    • 「保存する」ボタンをクリック（#btn-save）
    • [data-status="success"] を検知して完了確認
    • 次の行へ

5. 演習3: 複数ECサイトの一括処理

目的

条件分岐とループ処理を組み合わせた複雑なワークフローの習得

シナリオ

楽天・Amazon・Yahoo!の3サイトからそれぞれCSVをダウンロードし、販売管理システムに一括インポートします。

フローチャート

開始
  │
  ├─→ サイトリスト = [楽天, Amazon, Yahoo]
  │
  └─→ 各サイトに対して繰り返し:
        │
        ├─→ サイト選択画面を開く (http://localhost:8080)
        │
        ├─→ 対象サイトのリンクをクリック
        │     ├─ 楽天: #link-rakuten
        │     ├─ Amazon: #link-amazon
        │     └─ Yahoo: #link-yahoo
        │
        ├─→ CSVダウンロード画面へ移動
        │     ├─ 楽天: #nav-download → download.html
        │     ├─ Amazon: #nav-reports → reports.html
        │     └─ Yahoo: #nav-download → download.html
        │
        ├─→ ダウンロード条件を設定
        │
        ├─→ CSVダウンロード実行
        │
        ├─→ 販売管理システムでインポート
        │     └─ インポート形式を適切に選択
        │
        └─→ 次のサイトへ
  │
終了

各サイトのCSVダウンロードボタン

サイト ダウンロードボタンID インポート形式
楽天 #btn-download-csv 楽天RMS形式
Amazon #btn-download-csv Amazon形式

6. 演習4: 注文ステータス更新

目的

検索 → 詳細確認 → データ更新のフローの習得

シナリオ

「新規受付」ステータスの注文を検索し、内容を確認した上で「処理中」に更新します。

操作手順

Step 操作内容 対象要素
1 注文一覧画面を開く #nav-list
2 ステータスで「新規受付」を選択 [data-rpa-field="search-status"]
3 検索ボタンをクリック #btn-search
4 検索結果の1件目の「詳細」をクリック [data-rpa-field="detail-btn-0"]
5 モーダルで「編集」をクリック #btn-modal-edit
6 ステータスを「処理中」に変更 [data-rpa-field="status"]
7 「保存する」をクリック #btn-save
8 完了確認（自動的に一覧画面に戻る） [data-status="success"]
注意: 検索結果が複数ある場合は、ループ処理で全件更新することも可能です。その場合、更新後に再度検索を実行して次の対象を取得してください。

7. RPA対応属性一覧

要素識別用属性

属性 用途 例
id 要素の一意識別子 btn-download-csv, nav-input
data-rpa-field 入力フィールドの意味を明示 customer-name, order-id
data-status 処理状態の機械可読表現 ready, processing, success, error

処理状態（data-status）の値

値 意味 RPAでの使用例
ready 入力待ち・準備完了 フォーム入力開始の判定
processing 処理中 待機（次の操作を待つ）
success 処理成功 次のレコード処理へ進む
error エラー発生 エラーハンドリング実行

ダミーデータ生成機能

モックECサイトのCSVダウンロード画面では、毎回異なるダミーデータが自動生成されます。生成されるデータには以下が含まれます：

  • ランダムな日本人名（姓名30種類以上）
  • 全国47都道府県の住所
  • 5カテゴリ25種類の商品（食品、家電、衣類、日用品、美容）
  • 8種類の決済方法
  • 6種類の配送方法
ヒント: モック画面の「サンプルデータ生成」ボタンを押すと、表示データを更新できます。これにより、異なるパターンのデータでRPA処理をテストできます。
